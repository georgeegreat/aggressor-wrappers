feat(runners): add local execution backends for ArchCandy, Cross-Beta, TANGO,
 AmyloGram and AmyloDeep

Several predictors exist as local tools as well as (or instead of) web services,
but nothing in the package could execute them: the `_local` keys in the
amyloscope config are *parser adapters*, fed by files produced by hand outside
the pipeline. This adds local execution, plus the mechanism to choose between
local and web without waiting on a dead service to find out.

core/backend.py — backend selection

  Per-predictor policy `backend = auto|local|web` (also readable from
  AGGRESSOR_<KEY>_BACKEND):

    auto   run the local tool if its binary/JAR resolves, else the web service
    local  force local; a clear error if absent, never a silent fall-back
    web    the previous behaviour

  The decision uses shutil.which() / Path.exists() BEFORE any network call. It
  deliberately does not probe whether the service is up, because that probe is
  the slow flaky thing the fallback exists to avoid: a dead service should cost
  nothing, and it only costs nothing if it was never contacted.

runners/archcandy_local.py, predictors/archcandy_local.py

  ArchCandy has a working headless CLI, which is not obvious from the
  distribution. `java -jar` reaches fr.cnrs.crbm.ArchCandyFactory.Final, which
  builds a JFrame regardless of arguments and dies with HeadlessException;
  AnalysisFactoryCommandLine sounds right but is a 481-byte stub with an empty
  main that exits 0 and does nothing. The real entry point is
  MakeAnalysisWithoutApplet, invoked with -cp and twelve positional arguments.

  Three input traps, all found by running it:
    - -cp + class, never -jar;
    - headers must be sp|acc|entry — FastaParse.getEntryName splits on '|' and
      reads index 2, so >RPL27 aborts with ArrayIndexOutOfBoundsException;
    - a description after the entry name is also mandatory — >sp|X|Y raises
      StringIndexOutOfBoundsException, >sp|X|Y anything parses.
  The runner rewrites headers accordingly and maps the output folders (which
  embed the header verbatim) back to the original ids.

  The CANDIDATES DETAILS table is fixed-width and its Arc Type field contains
  spaces ("6 Res Arc 1"), so splitting on whitespace silently shifts every later
  column; spans are derived from the header line instead. Start/Stop are 1-based
  inclusive. "No candidates" returns an empty frame rather than raising, so one
  clean protein cannot drop the predictor for the whole panel.

  Verified against ArchCandy's own summary statistics: it reports 40
  amyloidogenic residues and a highest score of 0.665; the runner produces 40/99
  flagged and peak 0.665.

  Default score_mode is 'highest' rather than the web default 'cumulative': the
  test sequence produced 16 overlapping candidates, many sharing start=30 and
  each scoring ~0.6, which sum to ~9.6 against ArchCandy's own 0.57 threshold.

runners/tango.py, predictors/tango.py

  TANGO is licensed, platform-specific and has no web service, so it is
  local-only and its tests skip when the binary is absent.

  Long output paths fail SILENTLY: ~30 characters works, ~42 produces no file,
  no message and exit code 0. This surfaced only under pytest's deep tmp_path
  directories. The runner therefore chdirs into the output directory and passes
  a bare basename, so path length cannot matter.

  Aggregation is the score column (the quantity the literature thresholds, at
  5%); Beta, Turn, Helix and Conc-Stab_Aggregation are retained as aux channels
  rather than dropped — the secondary-structure propensities are what make TANGO
  more than an aggregation scale, since a segment predicted beta-aggregating and
  beta-structured is a different claim from one aggregating in a helical context.

  The parser rejects a mismatched sequence: TANGO echoes the input in its aa
  column, so a disagreement means the file describes a different protein, and
  reusing a stale output would misalign every position.

  Verified on Abeta42 at the 5% cut: 17-21 LVFFA (peak 86.9%) and 29-41
  GAIIGLMVGGVVI (peak 98.6%) — both canonical APRs, from TANGO's own numbers.

runners/amylogram.py, predictors/amylogram.py

  AmyloGram returns one probability per query, so joining a positional consensus
  needs an explicit modelling step. AmyloGram is an n-gram model trained on
  hexapeptides (AmyLoad), so whole-protein input is outside its fitted regime;
  sliding its training unit is closer to the training distribution, not merely
  more convenient. Defaults: window=6, aggregation=max, threshold=0.5.

  'max' rather than 'mean' because mean dilutes a single strongly amyloidogenic
  hexapeptide against weaker overlapping windows and flags strictly fewer
  residues (asserted in tests) — the same argument that makes 'highest'
  preferable to 'cumulative' for ArchCandy. Both are available and the choice is
  recorded in metadata, so a profile is never ambiguous about how it was made.

  Sequences shorter than the window are scored as one peptide rather than
  dropped: silently returning an empty profile would remove the tool from the
  consensus without saying so. The R side stays thin — the windowing and the
  projection live in Python where they are testable.

runners/amylodeep.py, predictors/amylodeep.py

  AmyloDeep positions are 0-based where the rest of the package is 1-based; read
  verbatim its profile sits one residue toward the N-terminus of every other
  tool, which is small enough to survive inspection and large enough to break a
  consensus requiring 80% window overlap.

  Rows may also be windows rather than residues: fewer rows than sequence_length
  implies a window of sequence_length - n_rows + 1, and position is a window
  start. A 31-residue sample produced 22 rows, implying a window of 10; read
  verbatim, residues 23-31 would have been left at zero. The parser detects
  either case and records granularity, window_size and position_base rather than
  assuming.

  Two operational failure modes are detected and reported with the fix named:
  the first run downloads model weights from huggingface.co (air-gapped setups
  need HF_HOME pre-populated), and jax_unirep imports pkg_resources, which
  setuptools >= 81 removed.

runners/crossbeta_local.py, predictors/crossbeta_local.py

  Drives a local checkout of cross-beta-predictor. Three properties of that
  script shape the runner, all read from its source:

  Output ignores the working directory: both the model path and the results path
  are resolved against path.dirname(__file__), so -o names a file, not a
  location, and results always land in <repo>/results/<name>.csv. The runner
  collects from there into the pipeline's work directory.

  That fixed location is a concurrency hazard introduced by our own scheduler
  change: two predictors writing results/prediction_result.csv would race and
  one would silently parse the other's output. Each invocation gets a unique
  output name.

  IUPred3 is a separate academic-licensed dependency; without it the script
  fails at import. The runner checks for it up front and names the download page
  instead of surfacing a traceback.

  The per-residue file is semicolon-delimited, and Amino_acids_score is a
  stringified list of single-key dicts keyed by the amino acid. That carries
  residue identity, so the parser uses it as an integrity check against the
  input FASTA - the same guard as TANGO. AR_position is the literal None when no
  region was found, which is a biological result and yields an empty region list
  rather than an error.

core/compat.py

  jax_unirep calls jnp.clip(x, a_min=-88). NumPy 2.0 renamed clip's bounds to
  min/max, JAX followed and removed the keyword, so every call raises TypeError
  deep inside mLSTMCell and AmyloDeep's UniRep model is unusable. Exactly one
  call site.

  Editing site-packages fixes it until the next environment rebuild, and leaves
  the environment unreproducible from its own spec. A vendored fork as a
  submodule would not take effect: jax_unirep is a transitive dependency
  resolved from the package index, not from this repository. Pinning an older
  JAX is unavailable - the a_min spelling disappeared in the same era as NumPy
  2.0, and those builds have no wheels for Python 3.12-3.14.

  So: a narrow runtime patch that probes for the fault, no-ops when the
  dependency is already correct, is idempotent, and warns once. It is
  numerically identical to the pre-NumPy-2 behaviour and additionally honours
  clip_value, which the original accepts but ignores.

  Because AmyloDeep runs as a subprocess - usually in its own conda environment,
  where this package may not be importable - the fix has to cross a process
  boundary. The runner generates a self-contained bootstrap that applies the
  patch and calls amylodeep.cli.main in the same interpreter that runs the
  model. Disable with use_compat_shim=false once the fix lands upstream, which
  is where it belongs.

Licensing

  PATH (Modeller + PyRosetta), TANGO, ArchCandy and Cross-Beta's IUPred3
  dependency are all separately licensed and non-redistributable. None of their binaries
  are vendored; each is located by config path, kept out of the default
  predictor list, and skipped cleanly when absent so CI stays green.

Depends on 'feat(schema): preserve auxiliary per-residue and region-level
predictor output' — the TANGO and ArchCandy runners emit aux channels.
