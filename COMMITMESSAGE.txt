feat(CrossBeta): add local execution backends for Cross-Beta

CrossBeta Predictor also exists as local tools as well as (or instead of) web services,
but nothing in the package could execute them: the `_local` keys in the
amyloscope config are *parser adapters*, fed by files produced by hand outside
the pipeline. This adds local execution, plus the mechanism to choose between
local and web without waiting on a dead service to find out.

runners/crossbeta_local.py, predictors/crossbeta_local.py

  Drives a local checkout of cross-beta-predictor. Three properties of that
  script shape the runner, all read from its source:

  Output ignores the working directory: both the model path and the results path
  are resolved against path.dirname(__file__), so -o names a file, not a
  location, and results always land in <repo>/results/<name>.csv. The runner
  collects from there into the pipeline's work directory.

  That fixed location is a concurrency hazard introduced by our own scheduler
  change: two predictors writing results/prediction_result.csv would race and
  one would silently parse the other's output. Each invocation gets a unique
  output name.

  IUPred3 is a separate academic-licensed dependency; without it the script
  fails at import. The runner checks for it up front and names the download page
  instead of surfacing a traceback.

  The per-residue file is semicolon-delimited, and Amino_acids_score is a
  stringified list of single-key dicts keyed by the amino acid. That carries
  residue identity, so the parser uses it as an integrity check against the
  input FASTA - the same guard as TANGO. AR_position is the literal None when no
  region was found, which is a biological result and yields an empty region list
  rather than an error.

core/compat.py

  jax_unirep calls jnp.clip(x, a_min=-88). NumPy 2.0 renamed clip's bounds to
  min/max, JAX followed and removed the keyword, so every call raises TypeError
  deep inside mLSTMCell and AmyloDeep's UniRep model is unusable. Exactly one
  call site.

  Editing site-packages fixes it until the next environment rebuild, and leaves
  the environment unreproducible from its own spec. A vendored fork as a
  submodule would not take effect: jax_unirep is a transitive dependency
  resolved from the package index, not from this repository. Pinning an older
  JAX is unavailable - the a_min spelling disappeared in the same era as NumPy
  2.0, and those builds have no wheels for Python 3.12-3.14.

  So: a narrow runtime patch that probes for the fault, no-ops when the
  dependency is already correct, is idempotent, and warns once. It is
  numerically identical to the pre-NumPy-2 behaviour and additionally honours
  clip_value, which the original accepts but ignores.

  Because AmyloDeep runs as a subprocess - usually in its own conda environment,
  where this package may not be importable - the fix has to cross a process
  boundary. The runner generates a self-contained bootstrap that applies the
  patch and calls amylodeep.cli.main in the same interpreter that runs the
  model. Disable with use_compat_shim=false once the fix lands upstream, which
  is where it belongs.
